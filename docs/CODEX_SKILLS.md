## Quando usar

- `uncodixfy`: melhorias de UI, layout, componentes compartilhados, refinamento visual e implementacao de frontend neste repositorio
- `frontend-skill`: suporte geral para desenvolvimento frontend moderno
- `figma-use`: leitura e exploracao de arquivos e especificacoes vindas do Figma
- `figma-implement-design`: traducao de design para codigo com maior fidelidade
- `playwright`: validacao funcional e navegacao automatizada
- `screenshot`: validacao visual rapida de telas e estados
- `gh-fix-ci`: apoio para diagnostico e correcao de pipelines
- `security-best-practices`: endurecimento de seguranca em implementacoes
- `sentry`: investigacao de erros e observabilidade, quando o projeto usar essa integracao
- `frontend-design`: UI diferenciada e de alto acabamento, evitando visual generico de IA
- `theme-factory`: exploracao de temas, paletas e direcao visual mais forte
- `webapp-testing`: automacao de validacao de UI via Playwright para apps locais
- `mcp-builder`: construcao de ferramentas e integracoes MCP para expandir o agente
- `brand-guidelines`: trabalhos guiados por identidade visual e consistencia de marca
- `ui-ux-pro-max`: base extensa de estilos, paletas, tipografia, heuristicas UX e recomendacoes por stack
- `get-shit-done (GSD)`: camada de context engineering, spec-driven development e execucao orientada por fases

## Skills mais insanas para este projeto

Se a meta for extrair o maximo da IA neste repositorio, a combinacao mais forte hoje fica assim:

1. `uncodixfy` + `ui-ux-pro-max` para concepcao e refinamento de UI
2. `frontend-design` + `theme-factory` para direcao visual forte
3. `playwright` + `webapp-testing` + `screenshot` para validacao de interface
4. `mcp-builder` + `openai-docs` para evoluir integracoes e ferramentas do agente
5. `get-shit-done (GSD)` para reduzir perda de contexto e organizar execucao grande

Quando a demanda for de frontend ou UI, priorize `uncodixfy`, `ui-ux-pro-max`, `frontend-design` e as skills visuais e de validacao conforme necessario.
